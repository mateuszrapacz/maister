TL;DR

Jedno kanoniczne źródło 28 ról zasila wersjonowany IR, manifest i deterministyczny projector. Codex używa generic prompt injection, Cursor natywnych Markdownów, a Kiro natywnych par JSON + prompt. Exact resolver działa fail-closed, a advisor nie ma żadnej specjalnej ścieżki.

Key Decisions

  • Portable core + ports and adapters + transactional staged projection.
  • Jedno źródło zachowania i dokładne maister:<role_id>.
  • Clean install: legacy Codex TOML znika bez migracji runtime.
  • Materialization, discovery i invocation pozostają osobnymi dowodami.

Open Questions / Risks

  • Codex spawn/model control, Cursor precedence i Kiro identity observation wymagają host-versioned probes.
  • Kiro multi-root transaction wymaga pełnego rollback i race coverage.
  • Upgrade środowisk z legacy TOML jest celowo poza zakresem.
