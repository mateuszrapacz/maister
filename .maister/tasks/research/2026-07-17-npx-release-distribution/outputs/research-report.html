TL;DR

Add a thin public @mateuszrapacz/maister npm launcher and keep GitHub Release archives as the distribution source of truth. The launcher resolves the target/version, downloads and verifies maister-<target>.tar.gz, extracts it privately, and delegates to the existing transactional installer. This enables npx --yes @mateuszrapacz/maister@latest install --target codex without manual extraction or a second host installer. Exact versions are reproducible; latest is convenience-only and must report its resolved release.

Key decisions

  • Use GitHub mateuszrapacz/maister and npm scope @mateuszrapacz.
  • Keep maister-install.mjs as the only target mutation authority.
  • Bind target, release, asset, digest, source manifest, and E3 evidence before installation.
  • Migrate Skillpanel distribution metadata separately from legal LICENSE attribution.

Open questions / risks

  • Latest-versus-exact-version default.
  • Safe cross-platform archive extraction.
  • Unsigned checksums improve integrity checking but do not authenticate the publisher.
  • The repository lacks npm package metadata and an npx entrypoint.

Executive answer

npx should be an invocation mechanism, not the place where plugin files are installed. The npm package hides download, checksum validation, extraction, and source-manifest handling, then invokes the archive's copy of maister-install.mjs with a temporary local: source.

npx package → GitHub Release archive → existing installer → host target

This preserves receipts, journals, locks, drift detection, rollback, recovery, and settings ownership.

Recommended architecture

LayerAuthority
npm launcherDownload, release resolution, archive validation, extraction, cleanup
GitHub ReleaseTarget archives and release metadata
Existing installerHost mutation, settings, receipts, journals, rollback, recovery

The launcher must not directly write .codex, .cursor, .kiro, receipts, journals, backups, or managed settings.

Release resolution

For exact 2.2.1, use tag v2.2.1 and the allowlisted asset:

TargetAsset
codexmaister-codex.tar.gz
cursormaister-cursor.tar.gz
kiro-climaister-kiro-cli.tar.gz

For latest, resolve the latest published non-draft/non-prerelease release, print the resolved tag, and apply the same asset rules. Never fall back to the default branch. GitHub exposes asset URLs, names, sizes, states, and digests through the Releases API and release assets API.

Verification and extraction

  1. Validate target/version.
  2. Fetch archive and metadata with bounded timeout/size.
  3. Compare archive SHA-256 with release evidence.
  4. Reject traversal, absolute paths, unexpected symlinks, and bad topology.
  5. Extract to a private temporary directory.
  6. Verify .maister-source.json commit/version/content hash.
  7. Verify fresh, matching E3 attestation.
  8. Invoke the existing installer only after all checks pass.

Current SHA256SUMS, SBOM, and PROVENANCE are unsigned. They detect mismatched bytes but do not authenticate the publisher; preserve this limitation in the wrapper's output and docs.

User experience

npx --yes @mateuszrapacz/maister@latest install --target codex
npx --yes @mateuszrapacz/maister@2.2.1 install --target codex
npx --yes @mateuszrapacz/maister@latest update --target codex
npx --yes @mateuszrapacz/maister@latest verify --target codex

The target should remain explicit because one package serves three hosts. The launcher forwards the installer's JSON and exit code.

Required tests

  • Target/asset allowlist and exact/latest resolution.
  • Network status, timeout, size, digest, archive traversal, symlink, and cleanup behavior.
  • Manifest/source commit/version/content-hash and E3 mismatch behavior.
  • Byte-identical target/settings state on every pre-install failure.
  • Fixture-based install/update/verify/status/uninstall for all targets.
  • Isolated-home npx smoke and release/npm version alignment.

Ownership migration

Update README source examples, Codex plugin author/developer metadata, maintained user documentation, npm metadata, and release URL construction from Skillpanel to mateuszrapacz. Handle LICENSE attribution separately as a legal decision.

Official npm references: npm exec/npx, package.json/bin, scopes, and publishing scoped public packages.

Recommendation

Implement the thin @mateuszrapacz/maister launcher first, without changing the transaction core or target overlays. Make exact version selection and full verification the baseline, then add latest convenience resolution.
