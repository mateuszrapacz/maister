Implementation Plan: First-Class Pi Support

TL;DR

Implement Pi as a fourth first-class Maister target through the existing target, overlay, materializer, transaction, runtime, evidence, and release seams. The result is one generated package under <agentRoot>/maister, a closed projection of 29 skills, 14 commands, and 28 agent descriptors, one identity-managed settings.json:packages[] member, and a pi.native adapter over public pi-subagents/delegation v1.

Work proceeds in dependency order: target/overlay closure, deterministic projection, managed-array transactions, public runtime/events, evidence, current four-target release admission, and finally the integrated lifecycle. Fresh E5/E6 passes promote the claim to pi.native-semantic; otherwise the release remains explicitly provisional or unavailable.

Key Decisions

Canonical behavior remains in plugins/maister/; Pi output is generated only in materializer staging. Target identity is pi and adapter identity is pi.native.
Skills become skills/**, commands become prompts/**, and agents become agents/maister-<role>.md. No Pi agents manifest key, generated commands/**, private import, or bundled pi-subagents is allowed.
Maister owns only the generated package tree and one normalized identity in settings.json:packages[]. Unrelated entries, fields, order, modes, symlinks, and topology remain operator-owned.
Current release admission is exactly [codex, cursor, kiro-cli, pi]. Historical three-target parity is no longer an active gate.

Open Questions & Risks

  • E1 must verify the actual Pi 0.80.10 ExtensionAPI/EventBus signatures.
  • E5 must prove package-agent discovery, sourceInfo, exact identity, and collision rejection for all 28 roles.
  • Process loss after started is terminal process_lost; retry uses a new dispatch ID and is never resume.
  • The managed-array ownership primitive must cover receipts, drift, rollback, recovery, and fault injection without touching unrelated packages.
  • Every changed historical parity assertion is recorded in implementation-log.md with a replacement current-state proof.

Task Groups

G1 — Register Pi and close the overlay contract

Owned files: lib/distribution/targets.mjs, target-paths.mjs, overlay-loader.mjs, overlays/schema/overlay-v1.schema.json, overlays/pi/overlay.yml, overlays/pi/inventory.yml, Pi overlay fixtures, and Pi cases in target-registry.test.mjs/overlay-contract.test.mjs.

  1. Add pi, pi.native, target paths, PI_CODING_AGENT_DIR precedence, package containment, POSIX-only support, settings path, managed root, probes, and six semantic bindings.
  2. Extend strict overlay validation for managed_array_entries, closed inventories, forbidden topology, compatibility tuple, and command/role maps.
  3. Add acceptance and rejection fixtures for unknown fields, unsafe paths, duplicate destinations, incomplete roles, missing bindings, and bundled prerequisites.

Proof: target registry and overlay tests pass; E1 binds registry, overlay, and schema digests. No Pi path escapes agentRoot.

G2 — Closed projection and deterministic package materialization

Owned files: agent-projector.mjs, agent-projection-validator.mjs, materializer.mjs, provenance.mjs, new pi-command-projection.mjs, Pi cases in agent-projection.test.mjs/source-materializer.test.mjs, and new pi-package-projection.test.mjs fixtures.

  1. Read canonical 29 skills, 14 commands, and 28 agents; emit skills, prompts, and maister:<role> descriptors exactly once with source/projection digests.
  2. Generate the exact package manifest, agent-projection-v1.json, pi-command-projection-v1.json, .maister-source.json, extension entry point, and overlay-listed runtime closure.
  3. Reject omissions, duplicates, stale hashes, shadowed origins, unsafe outputs, wrong identities, symlinks, node_modules, credentials, sessions, other targets, and external pi-subagents.
  4. Fix LF/UTF-8 encoding, bytewise order, inventory modes, package-root mode, and source/date provenance.

Proof: the Pi package projection suite proves exact counts, manifest shape, forbidden closure, and double-materialization byte identity. E2/E3 bind package, projection, source tree, and portable closure.

G3 — Managed-array ownership and transactional lifecycle

Owned files: settings-owner.mjs, transaction-manager.mjs, receipt-schema.mjs, journal-schema.mjs, drift-detector.mjs, recovery.mjs, path-safety.mjs, existing installer/crash tests, and new pi-managed-array.test.mjs.

  1. Parse settings strictly under the existing lock and normalize string/object sources to one contained NFC realpath identity.
  2. Preserve matching representation and opaque fields; reject malformed/non-array settings, duplicate identities, conflicting filters, disabled autoload for native claims, symlink escapes, drift, and concurrent writes.
  3. Create minimal settings only when absent, change only the owned member, and record managed_array_entries_v1/pi_local_package_v1, before/after hashes, unmanaged projection, modes, and backup.
  4. Stage and commit package tree before settings member; verify both; roll back both on failure and retain backups for unresolved recovery.

Proof: string/object, duplicate/filter, drift, mode/order, concurrent writer, crash, and injected-failure tests compare bytes, modes, symlinks, existence, and topology. E4 is blocked until the full lifecycle is green.

G4 — pi.native, public delegation v1, and durable observations

Owned files: new host-adapters/pi-native.mjs; host-adapter index; production-runtime.mjs, agent-resolver.mjs, dispatch-agent.mjs, dispatch-contract.mjs; execution event schema/payload/writer; new canonical lib/distribution/pi-extension-source.mjs; adapter/runtime/event tests; and new pi-native-adapter.test.mjs.

  1. Resolve only exact receipt-backed maister:<role> IDs and freeze source digest/execution profile.
  2. Import only named public delegation v1 exports. Build bounded requests with requestId === dispatch_id, subscribe before emit, and implement exact status/cancel/retry mapping.
  3. Write dispatch_requested before side effects; sanitize updates; require exact terminal response.agent; classify malformed, wrong, missing, cancelled, timed-out, budget, and process-lost outcomes.
  4. Implement maister-observation-v1 JSONL: canonical JSON, sequence, SHA-256 chain, fsync, redaction, bounds, and read-time continuity checks.
  5. Register exactly one maister-delegate command and use only public Pi ExtensionAPI/EventBus plus shutdown cleanup.

Proof: fake-public-module adapter tests cover subscriptions-before-emit, identity, cancellation, process loss, retry IDs, and bounded events. The fresh pinned-host probe now passes E5/E6 through the real public boundary.
