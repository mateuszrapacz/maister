Specification: First-Class Pi Support

TL;DR

Add Pi as a fourth first-class Maister target through one deterministic generated package, a closed hybrid projection, one identity-managed packages[] member, and a thin public pi-subagents/delegation v1 adapter.

The normative contract is complete in spec.md Appendix A. A complete bound E1-E6 manifest publishes pi.native-semantic; when both native levels are unavailable, structural/transactional packaging remains explicitly provisional.

Decisions

Target identity is pi / pi.native. Current release admission is exactly Codex, Cursor, Kiro CLI, and Pi; historical parity machinery is removed from active code and metadata.
The package is generated at <PI_CODING_AGENT_DIR>/maister, defaulting to ~/.pi/agent/maister. Pi, pi-subagents, settings outside one member, auth, trust, sessions, and unrelated packages remain operator-owned.
All 29 canonical skills, 14 canonical commands, and 28 canonical agents are represented exactly once. Skills remain skills, commands become prompts, and one generated extension contains only imperative registration/delegation.
Settings ownership is versioned managed_array_entries_v1 / pi_local_package_v1. The public runtime boundary is versioned delegation protocol v1 and Maister owns the durable observation stream.

The manifest has only name, version, private, description, and Pi extensions/skills/prompts fields. Agent discovery is performed by the public pi-subagents package contract, not by inventing a Pi agents manifest key. Generated output is LF, bytewise sorted, deterministic, symlink-free, and contains no credentials, sessions, auth, node_modules, or bundled pi-subagents.

Settings contract

settings.json is <agentRoot>/settings.json; only packages[] is managed. Strings and objects with source normalize to one absolute NFC path identity. Existing representation and unrelated fields/order are preserved; duplicate identities, conflicting filters, malformed JSON, non-array packages, escaping symlinks, drift, and concurrent writes fail before mutation.
Missing settings creates a minimal file; a missing packages property is added; a non-array property is rejected. Receipts record ownership/merge schema versions, normalized identity, before/after hashes, unmanaged projection hash, modes, and backup reference. Rejected or rolled-back changes restore bytes and topology exactly.

Runtime and durable events

The adapter imports only the public pi-subagents/delegation export and uses these exact v1 event constants:

prompt-template:subagent:request
prompt-template:subagent:started
prompt-template:subagent:update
prompt-template:subagent:response
prompt-template:subagent:cancel

It freezes the role identity from Maister's receipt, subscribes before request emission, writes dispatch_requested before side effects, bounds/sanitizes updates, and requires exact terminal response.agent equality for candidate success. Duplicate IDs, missing terminal responses, wrong identity, malformed events, exit 0 without response, cancellation uncertainty, and process loss are typed failures; retry uses a new dispatch ID.

maister-observation-v1 JSONL records use sequence numbers, event type, dispatch/request IDs, requested/observed identity, status, bounded payload, previous hash, and SHA-256 hash over canonical JSON. The writer fsyncs each append and verifies chain continuity. Raw transcripts, auth, cookies, and unbounded tool arguments/output are never persisted.

Lifecycle, recovery, and traceability

Install locks the Pi agent root, stages on the same filesystem, commits the package tree before the one settings member, verifies receipt/discovery, and journals every boundary. Rollback preserves backups; unresolved recovery is never silently overwritten. The deterministic archive fixes source-date epoch, path order, modes, uid/gid, gzip timestamp, and external prerequisite metadata.
