Pi support guide

Graduated support: A complete bound E1-E6 manifest publishes pi.native-semantic. If both native levels are unavailable, the explicit fallback is pi.structural-transactional.provisional. The current pinned host has passed E5/E6.

Supported contract

Pi 0.80.10, Node 25.9.0, pi-subagents 0.35.1, and public delegation protocol v1. Unsupported tuples fail closed. Operator-owned Pi state and external prerequisites are not bundled.

Install

node plugins/maister/bin/maister-install.mjs install \
  --target pi \
  --source local:/path/to/extracted-package \
  --evidence plugins/maister/.maister-e3-attestation.json \
  --json

Lifecycle

Use verify, update, rollback, recover, and uninstall through the same generated installer. Recovery selects an exact journal with --journal-id when multiple unresolved journals exist.

Ownership and delegation

Maister owns only its package tree and one identity-managed packages[] entry. The public maister-delegate extension command delegates through public v1 exports with bounded budgets, typed cancellation/process-loss outcomes, and durable redacted hash-chained observations.

Evidence

E5 proves the pinned host and public package identity. E6 proves the full ordinary/advisor delegation lifecycle. Both must pass together on one fresh binding; unavailable, mixed, stale, or expired records cannot publish a native-semantic claim.
