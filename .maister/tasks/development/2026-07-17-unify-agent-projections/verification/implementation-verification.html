Remaining findings

WarningRV3-W1 — stdin ingestion is not bounded and read failures can bypass the typed envelope

Location: plugins/maister/bin/maister-agent-gate.mjs:19-22; promises at README.md:167,186.

The default input = fs.readFileSync(0) runs before the function body and before its try. The entire stream is allocated before the 1 MiB length check, and a read failure can skip the documented one-envelope failure path.

Recommendation: acquire stdin inside try, read incrementally with a hard retained cap of MAX_INPUT_BYTES + 1, stop immediately on overflow, and add oversized subprocess plus injected read-error regressions that assert exit 2 and one E_AGENT_OWNER_INPUT envelope.

WarningRV3-W2 — optional native cancellation lacks a closed v1 callback contract

Locations: README.md:195; host-adapters/exact-native.mjs:37,57,59,67,69.

Optionality and best-effort semantics are correct, but the public contract does not define the request shape, exact post-side-effect durability-failure trigger, or callback return/throw mapping. Code currently passes either a launch observation or an ad-hoc {error} mapping.

Recommendation: publish one closed request v1 with dispatch/native identity, reason, recording phase, and observation identifiers; define timing and outcome mapping; add a public bridge contract test.
