TL;DR

Maister is never published to npmjs, GitHub Packages, or another registry. Users acquire and run the repository package through npm install or npm exec from an exact GitHub tag or full commit.

The launcher consumes a prepare-generated checkout-commit manifest, acquires exact GitHub Release assets anonymously or through a narrowly bounded private API credential path, verifies and streams them into a private operation root, and delegates to the release installer. GitHub Release archives remain payload authority; maister-install.mjs remains sole mutation authority.

Key Decisions

  • Exact #vX.Y.Z tags and full commits only; no branch, default ref, latest, dist-tag, or semver range.
  • A fixed non-shell prepare producer records the actual checkout commit; protected CI separately proves the original exact selector.
  • private:true, no publishConfig, and exact normal dependency tar@7.5.20.
  • Public GitHub access emits no authorization; private API precedence is GH_TOKEN → GITHUB_TOKEN → bounded gh auth token → anonymous.
  • Authorization is confined to api.github.com; assets use API URLs and octet-stream negotiation, with auth stripped cross-host.
  • State-only commands are receipt-bound and launcher-network-free after package acquisition.
  • Release CI publishes GitHub Release assets only and has no registry publication command or credential.
  • The canonical repository is public now: real anonymous E2E is mandatory now, while real private E2E activates only after a visibility migration.

Open Questions / Risks

No critical or important design decision remains. Release stays NO-GO until protected-tag controls, current real public and hermetic private evidence, streaming RSS evidence, terminal aggregate transactions, abrupt-crash boundaries, and multiple-journal evidence are complete.

This TL;DR and every visual summary below are non-normative. The exact normative acceptance contract is the R-001–R-038 table; the Markdown twin carries the full normative detail.

1. Status and superseded scope

In scope

  • Exact Git-package npm install/npm exec.
  • Public/private GitHub Release API transport and assets.
  • Bounded credentials, streaming archive verification, private extraction.
  • Receipt-bound control planes, exact journal recovery, crash/transaction evidence.
  • GitHub-only Release CI, current real public E2E, hermetic private transport, and the future private-migration checklist.

Explicitly out

  • Any Maister registry publication, dist-tag, latest, range, or branch selector.
  • Arbitrary repository/URL/asset/endpoint or fallback version.
  • Second installer, launcher-owned host mutation, persistent cache, telemetry, or UI.
  • Unsupported hosts, marketplace lifecycle, or legal LICENSE attribution change.
Supersession: every historical npm publication, NPM_TOKEN, npm publish/view, dist-tag, propagation, registry smoke, exact npm publication, and latest-promotion assumption is historical only. npmjs is allowed solely for read-only acquisition of locked tar@7.5.20.

5. Exact candidate identity

Protected CI proves the original exact selector. A fixed non-shell prepare producer reads the actual checkout HEAD and atomically carries a restricted-mode .maister-resolved-commit.json; the launcher compares that commit with package version, Release tag target, sidecar, archive manifest/digest, E3, and installed identity. npm runtime metadata is not treated as a reliable selector source. Any absence, generation failure, or mismatch is terminal.

6. Public/private authentication

Anonymous-first

  1. Request the fixed exact-tag Release route without auth.
  2. Public success fixes anonymous mode and skips credential lookup.
  3. Only eligible 401/403/privacy-preserving 404 triggers credential resolution.
  4. Retry the same route once; never select another repo/ref.

Strict precedence

  1. GH_TOKEN
  2. GITHUB_TOKEN
  3. gh auth token --hostname github.com
  4. anonymous

Malformed explicit environment credentials fail closed instead of falling through.

Injectable credential command 
{
  "executable": "gh",

  "shell": false,
  "timeout_ms": 5000,
  "max_stdout_bytes": 16384,
  "max_stderr_bytes": 8192

Success requires exit 0, no signal/truncation, and one 1–4096-character printable non-whitespace token line. Output is bounded, memory-only, validated, fully redacted, and never persisted. Missing executable, non-zero, signal, timeout, overflow, or malformed command output deterministically yields no command credential and anonymous behavior.

7. API asset transport

Authorization is permitted only on approved HTTPS routes at exact host api.github.com in authenticated-private mode. Every archive and sidecar uses its Release API asset URL/ID with Accept: application/octet-stream. Direct 200 and bounded 302 are supported; authorization is permanently stripped before a cross-host redirect and never reaches browser/content hosts.

ResourceLimit
Release JSON / checksums1 MiB / 64 KiB

Redirects5
Pre-header attempt/body-idle15 seconds each
Metadata/archive/whole acquisition30 / 120 / 180 seconds

Each initial, redirect, or retry fetch gets a fresh 15-second pre-header clock from immediately before fetch through final headers, including DNS, TCP, TLS, proxy negotiation, and redirect headers. Resource and 180-second aggregate walls never reset. Expiry is typed, aborts, and cleans partial files; eligible pre-semantic failures may retry. Final headers start the separate 15-second body-idle timer.

8–9. Verification, archive safety, and bounded memory

Normative order: validate → resolve metadata → select assets/create temp root → stream sidecars/archive → compare digests → stream-inspect → reject unsafe archive → extract planned entries privately → no-follow revalidate source/overlay/installer/E3 → build descriptor → delegate.

Rejected archive behavior

  • Malformed/truncated headers, concatenated gzip, trailing data, unsupported extensions.
  • Absolute/drive/UNC/traversal/backslash/control paths and exact/case/Unicode collisions.
  • Links, devices, FIFO/socket/whiteout/sparse/unknown types.
  • Unsafe ownership/modes, file-parent conflicts, foreign target/topology.

Strict extraction

Exact tar@7.5.20, strict:true, no preserve paths/owner, no overwrite/follow, depth 128, ratio 100, metadata 1 MiB, and a filter admitting only planned regular files/directories into a new identity-captured root.

Archive propertyLimit
Entries / one file / expanded bytes50,000 / 128 MiB / 512 MiB
Path bytes / segments1,024 / 128
Compression ratio100:1 after first 1 MiB
Peak RSS over baseline≤128 MiB on Linux, macOS, Windows
RSS growth for ≥2x expanded fixture≤16 MiB

Download hashing, inspection, and extraction are backpressure-aware streams and never retain the full compressed or expanded archive. Byte caps without RSS/stream evidence are unavailable evidence.

10–13. Delegation, state, recovery, output, and zero mutation

Receipt-bound control plane

Install/update stage the verified installer/runtime closure in the same journaled transaction as candidate receipt and active pointer. Receipts bind contained refs, tree/installer hashes, CLI contract, and source identity. Referenced historical closures survive update/rollback/recovery; pruning is terminal, reference-aware, and journaled.

State-only discovery

No transport, credential command, Git, registry, PATH, cache, checkout, sibling receipt, or nearest-version fallback. No-follow active receipt/control-plane validation chooses one exact installer; legacy/ambiguous/corrupt state fails typed and mutation-free.

Journals and abrupt crashes

recover --journal-id <uuid> selects exactly one unresolved journal. Unqualified recovery with multiple journals fails closed. Abrupt termination is injected after every durable lock/journal/backup/staging/control-plane/receipt/pointer/verification/rollback/prune/terminal boundary, with exact byte/mode/link/existence/topology assertions.

Output, signals, cleanup

Before delegation, typed redacted diagnostics use stderr and stdout stays empty. After spawn, child streams, JSON, receipt/journal paths, exit codes 0 and 2–8, and signal outcome are authoritative and unwrapped. Signals abort/forward/wait; POSIX re-raises when possible and Windows maps 130/143. Exact-root cleanup never touches installer state or replaces the child result.

Terminal aggregate gate: the exact candidate transaction suite must explicitly pass, fail, or time out with final-tree evidence. Silence, interruption, or partial focused runs are unavailable evidence.

14–16. GitHub-only release, smokes, and platform matrix

BUILT → GITHUB_PUBLISHED → GITHUB_VERIFIED
      → PUBLIC_NO_AUTH_SMOKE_VERIFIED
      → HERMETIC_PRIVATE_TRANSPORT_VERIFIED

Release CI may create, verify, and upload deterministic GitHub Release archives and sidecars only. It has no npm publish/view/dist-tag/deprecate/registry-observation step, no registry publication configuration, no NPM_TOKEN or equivalent publication credential, and no npm publication state. Static and network-observing checks prove zero registry mutation while allowing read-only locked dependency acquisition.

Public smoke

Clear auth overrides/user gh state; acquire exact public tag and commit; use anonymous metadata/API assets; prove no Authorization; exercise isolated install/status/verify/uninstall for all release-policy targets.

Private behavior now

Hermetic injected seams prove credential precedence, eligible retry, direct 200/302 API assets, auth confinement/stripping/redaction, failures, and no leakage without requiring another repository.

Future private migration

If the canonical repository becomes private, normative docs block release until Git credentials, environment/gh API credentials, protected secrets, authenticated canonical private Git and Release E2E, no token leakage/cross-host auth, and exact selector/manifest/tag/sidecar/archive/E3 evidence all pass. Real private E2E is mandatory only then.

Linux, macOS, and Windows run Node 22 and the declared npm major across CLI, auth, transport, archive/RSS, zero mutation, cleanup, signals, authority, transaction, crashes, journals, package, identity, and smokes. A platform semantic unavailable/skip is never a pass.
