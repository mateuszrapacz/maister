Implementation specification

Codex Maister lifecycle skills

Add bye, dev, next, resume, and status as Codex-only overlay skills, exposed as $maister:<command>.

Scope and architecture

The five SKILL.md assets belong in plugins/maister/overlays/codex/assets/skills/. The Codex overlay merges this tree into skills/. The common portable inventory, Cursor projection, Pi aliases, and removed host-specific builders remain unchanged.

Behavior

  • bye preserves active workflow state and produces a resumable handoff.
  • dev forwards the original input to $maister:development.
  • next suggests exactly one next action without executing it.
  • resume continues the matching workflow from orchestrator-state.yml.
  • status reports workflow state without starting or resuming work.

Acceptance and verification

Materialization must expose all five files with host-relative frontmatter names, valid modes, no forbidden vocabulary, deterministic output, and reconciled parity evidence. Focused overlay/materializer checks and the configured core, evidence, topology, projection, and release tests must pass.
