# Project Instructions Documentation Section Template

Add this section to the project's project-instructions file. Place it prominently near the top. Verify the project documentation index path is correct and the file exists before adding.

```markdown
## Project Documentation & Standards

Before writing or changing any code — even for quick, direct requests that do not go through a Maister workflow — ground yourself in the project's documentation:

1. Read @.maister/docs/INDEX.md to see what's documented. It is the map to everything the team maintains — coding standards by domain, project vision, technology, architecture, and other project knowledge.
2. Then open and read the specific files it points to that are relevant to your task — standards and project docs. The index alone is not enough.
3. Follow the standards as you work. If one conflicts with the task, ask the user.

### Standards Evolution

When you notice a recurring pattern that is not captured in standards, suggest adding it through the project standards workflow.

### Maister Workflows

When a Maister workflow is invoked, execute it through the skill loader immediately. Do not skip the workflow for a straightforward task.
```
